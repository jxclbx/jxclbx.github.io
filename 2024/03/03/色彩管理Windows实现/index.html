色彩管理Windows实现

jxclbx
技术 技术

发布于：Mar 3, 2024

由于我们在互联网中的几乎全部内容都是缩限到sRGB空间中的色彩，所以看到这篇文章时，你大可不必因为担心手机无法正确显示广色域内容，去更改手机显示的画面。其实手机的色彩管理远比你的PC（windows）要成熟。

我们用一张将饱和度降低了50%的位于Diaplsy-P3色域下的红色图标开启这个文章

这张图片中的图案以及背景的红色，都超过了sRGB的显示范围，在缩限时，均被归为sRGB的红色上限 (255, 0, 0)，这两种红被色彩管理转换为了sRGB的最红，也就是同一种红色，所以在sRGB下，这张图片无法看出图案。

由于饱和度的降低，广色域内容得以在sRGB之中显示。我们可以得出结论 虽然看起来像一句废话：更广的色域涵盖了更多的颜色，位于sRGB之外的颜色无法在sRGB屏幕被完全显示

为什么进行色彩管理？

  1. 打印输出与屏幕显示色差大：当一幅图像在计算机屏幕上看起来色彩饱和度高，细节清晰，但打印出来后颜色显得暗淡或失真，这就是没有进行有效的色彩管理。
  2. 跨设备色彩不一致：一个设计在不同的显示器上查看时，色彩出现明显差异，比如在一个显示器上是鲜艳的蓝色，在另一个显示器上却变成了暗淡的蓝色。
  3. 扫描与原件色彩不符：扫描器扫描出的图片与原始文件在色彩上有很大的偏差，没有保持原有的色彩信息。

色彩管理的目的是为了确保在不同的设备、软件和媒介之间，色彩能够尽可能一致地被传递和呈现，即使这些设备的色彩表现能力不同。色彩管理通过使用色彩配置文件（如ICC配置文件），来描述每个设备在色彩方面的特性，并通过这些信息来调整色彩，以便在不同设备之间实现色彩的一致性。

什么是色彩管理

Adobe官方解释为：一个可以在设备之间准确地解释和转换颜色的系统。

色彩管理系统 (CMS=Color Management System) 将创建了颜色的色彩空间与将输出该颜色的色彩空间进行比较并做必要的调整，使不同的设备所表现的颜色尽可能一致。Adobe还强调色彩管理不是色彩校正，所以不要以为色差出现都是因为设备的颜色不准确，最可能的原因是不了解色彩管理。

根据色彩管理概述，我们先详细解释这个过程，后边我将会更加通俗易懂地讲述。

  1. 表征（Characterize） ：每个色彩管理的设备（无论是输入设备如显示器，还是输出设备如打印机）都需要一个个性化的表格或“色彩配置文件”，这个配置文件描述了该特定设备的色彩响应。这意味着，每个设备都有其特定的方式来表现颜色，这需要通过色彩配置文件来定义。
  2. 标准化（Standardize） ：每个色彩配置文件将这些色彩相对于一个标准化的参考色彩集（即“配置文件连接空间”）来描述。这样做的目的是为了确保不同设备之间在色彩传递时有一个共同的参考标准，从而使色彩转换成为可能。
  3. 转换（Translate） ：色彩管理软件使用这些标准化的配置文件来将色彩从一个设备转换到另一个设备。这通常是通过色彩管理模块（Color Management Module, CMM）来完成的。色彩转换并不是一个简单的过程，尤其是当涉及到不同设备的色域不匹配（即所谓的“gamut mismatch”）时，CMM需要寻找最佳的近似方式来实现色彩转换。

举一个简单但不完全准确的例子来说，Adobe RGB和sRGB两个色彩空间，Adobe RGB拥有比sRGB更广的颜色范围，尤其在绿色和蓝色的色域上。考虑红色：Adobe RGB中的红色（例如，200,0,0）和sRGB中最鲜亮的红色（255,0,0）所匹配，此时我们需要将Adobe RGB中的红色坐标换算为sRGB中的红色坐标，来让广色域中的红色在sRGB空间正常显示。

色彩空间

首先，我们应该知道CIE (Commission Internationale de L’Eclairage) 设立的色彩空间的概念：

CIE色域是由国际照明委员会（CIE）定义的一种色彩空间，它包含了由实验得出的人眼可以感知的所有颜色。CIE色域通常在一个叫做色度图的二维平面上表示，这个平面由x和y坐标定义，每一种颜色都对应于色度图上的一个点，而白色则位于图的中心。

CIE色域包含了大部分其他色彩空间（如Adobe RGB）的颜色，任何在CIE色域内的颜色都可以通过某种方式在其他色彩空间中表示。然而，每种色彩空间都有其自己的限制，因此并非全部颜色都可以在任意一个色彩空间中准确地表示。同时，一些非可见光（位于CIExyz坐标外的颜色）如何在软件层面进行处理还不得而知。

CIE是如何得来的，我参考了这篇文档：色域马蹄图是怎么来的？——CIE 1931 XYZ色彩空间详解，基本可以解释色彩空间和色彩管理的源起。

ICC配置文件

ICC文档中提到

The information necessary to adequately define the PCS is contained in … The concept of a PCS is a vital element in the ICC architecture. It allows the profile transforms for input, display, and output devices to be decoupled from each other so that they can be combined as needed. A well-defined PCS provides the common interface for the individual device profiles as illustrated in… predictable results when applied to colour values.

PCS在ICC色彩管理体系中起到了枢纽的作用，是一个用于连接各种输入、显示和输出设备的色彩空间。ICC通过定义一个通用的PCS，不同设备的配置文件可以独立创建并在需要时组合使用，从而实现色彩信息在不同设备间的准确转换。

在ICC框架中，PCS通过将不同色彩空间转换到统一的坐标系中，允许输入、显示和输出设备的配置文件转换互相解。定义好的PCS为各个设备配置文件提供了一个共同的接口（CIExyz色彩空间），如图D.1中所示：PCS是源转换的虚拟目的地和目的地转换的虚拟源头。如果源和目的地的转换都基于同一个PCS定义，即使它们是独立不同时间创建的，它们也可以在运行时被颜色管理引擎配对，并在应用于色彩值时产生基于统一标准和可预测的结果。

未经色彩管理的错误情况

在搞清楚了原理之后，我来从认知色彩管理的时间和逻辑顺序讲述整个调整过程需要注意的内容

sRGB显示器 显示 sRGB内容

这种情况通常看起来是正确的，因为未经颜色管理的内容经过Windows系统的强制处理为sRGB内容输出，这在我看来是瞎猫碰死耗子的行为。

为什么这么说呢？因为哪怕是sRGB的显示器也会拥有其自己的色差，如果不进行色彩管理，显示器可能在sRGB色彩空间内出现各种偏移

这张图片为了模拟偏色的显示器，我将RGB曲线的红色通道偏移了5%。

实际你的显示器因为许多年没有维护、打开了护眼模式、脏了、自己瞎调色温，导致了许多显示的误差。即使显示器和图片都位于sRGB，也是需要色彩管理的。

更何况有些异形色域显示器，其显示色域不是标准的sRGB，其显示色域就好比广色域，从而带来了类似于广色域显示器的误差。

广色域显示器显示没有嵌入icc的sRGB内容

广色域看起来其实并没有大家想象的这么夸张。一张记录了P3色域的图片并不是说将所有的颜色变得更为“鲜艳”，而是说在保证记录下sRGB原图中应有的信息的同时，记录下额外红色区域的色彩，如下图所示：

也就是说，正常情况下，我们在正常管理颜色的广色域下面，看到的应该是一副与在sRGB显示器上外观几乎一样，仅接近最高饱和度的区域中的部分，这部分饱和度有所提高的图像。

而Windows未经色彩管理的情况下，会认为你正在使用的是一台sRGB显示器。在这种情况下，如果不进行色彩管理，操作系统中的应用程序内容与系统UI一起以一种类似于没有参考系的运动物体、没有坐标系的坐标一样投射到广色域的显示器上。

早期Windows系统的本意是所有人都不进行色彩管理，所有人使用的都是sRGB显示器，便省去了色彩管理部分的代码，也就是刚才所描述的添加参考系的过程。系统将红色饱和度最高的内容直接按照8bit能输出的最大值（255，0，0）来处理，绿色（0，255，0），蓝色同理，而并没有考虑Windows认为的“sRGB的最大值”，在广色域显示器上并不是“最大值”。这时候的问题我们可以抽象为：系统将sRGB色彩空间的所有内容拉伸到了一个更大的色域里，也即将系统整体显示的饱和度进行了提高，导致过饱和。这不是正确色彩管理流程希望看到的结果。

很高兴，Windows工程师慢慢注意到了这个问题，Windows10加入了HDR功能是基于BT.2020色彩空间，由系统自动管理色彩映射；Windows的照片查看软件也加入了对显示器icc文件的支持；Windows11 22H2版本也加入了 ACM功能 ……

sRGB显示器显示没有嵌入icc的广色域内容

最后一种情况便是拍摄了广色域内容，在未嵌入icc时呈现灰色。去除掉gamma曲线的不同，在色彩空间上两者也有较大差距：

可以看到，Sgamut3非常宽广，包含了许多sRGB（Rec.709）没有包含的色彩

由上一种情况，我们便可以推测：Windows将索尼拍摄的没有嵌入icc的原片认为是sRGB的内容，在索尼Sgamut 3色彩空间中原本非常饱和的各种颜色，如图：Sgamut3中的某个绿色RGB坐标可能为（40，150，27），这种绿色在CIExyz坐标中饱和度远高于sRGB的上限，可它却被映射到了甚至离sRGB上限都很远的地方。这种情况便会导致广色域内容在Windows系统下欠饱和。

加载校准文件

此时，我们需要知道，Windows系统支持色彩管理的一些软件：win10照片，Photoshop，Lightroom，在使用这三个软件的时候，三个软件会绕开Windows的逻辑，读取屏幕默认的配置文件，将嵌入了icc的图片转换到正确的显示上。也就是说，只要获得了色彩配置文件，这三个软件在不进行任何多余操作的情况下会自动进行色彩管理。也就是说，接下来介绍的许多让显示更正确的方法，只能在你不使用这些自带色彩管理的软件时使用。

DisplayCAL

DisplayCAL内置了屏幕校准的功能，但这种校准方式不是很适合广色域显示器的Windows，因为目前Windows许多系统资源本体没有内置色彩管理的功能。在广色域显示器上，Windows依然按照sRGB色彩空间的坐标系进行输出，而实际显示色彩是广色域的，相当于在一个“拉伸失真”后的显示上，再去进行校正，于事无补。

推荐在具有100%sRGB的显示器使用这个方法，但世界上没有完美的100%，所以都存在一些错误的拉伸。

Novideo_sRGB

我们无法修正Windows的错误，所以，在你需要浏览系统内容以及使用没有经过色彩管理的应用程序时，我们只能将广色域显示器“缩小”到sRGB色彩空间下进行显示，以获得最好的观感和准确性。

程序需要N卡支持，教程在此

Novideo规避掉了Windows强制sRGB的问题，可以很好的解决色域不匹配，我较为推荐这个软件。

如何用广色域显示器查看图片

Windows自带照片应用

注意，使用Windows自带照片应用时不要打开上述提到的色彩管理软件。

这个软件也是最简单的一个，你只需要将其拖动到对应的屏幕上，便可以获得一个完全正确的显示效果（前提是你的图片嵌入了icc并且你没打开全局的色彩管理）

Photoshop

Photoshop相对更为麻烦，涉及到许多参数的调整

颜色设置

  • RGB：该参数意味着Photoshop的工作色彩空间，为了获得作图时最佳的视觉效果，请选择你目前色彩空间最大的显示器的配置文件
  • 色彩管理方案：打开新图片选择保留嵌入的配置文件，粘贴图片到工程里选择转换为工作色彩空间
  • 如果你的图片嵌入了icc，那么在导入了这张图片后，Photoshop会自动将图片根据图片内置的icc和系统的显示器配置文件进行正确显示
  • 如果你的图片没有嵌入icc，你需要自己选择配置文件

指定配置文件

来到了很重要的一步，这步可以通俗地理解为在CIExyz坐标中，（255，0，0）这样坐标的落点，这个步骤直接告诉了Photoshop每个颜色值在绝对色彩、可见光范围上究竟该位于什么位置，这样我们才存在完全正确显示一个内容的可能性。

照片的导出

最后一步了！导出时有如下两个选项：

  • 强制转换为sRGB：这个方法基于一个假设，即所有的显示器都遵循sRGB标准，所有的图像内容也都是在sRGB环境下产生的。这种方法忽略了显示器之间的差异以及广色域内容的存在，在广色域显示器上，可能会导致色彩过饱和或失真。

  • 嵌入配置文件：相比之下，使用ICC配置文件的方法则更为科学和精确。每个设备的ICC配置文件都是该设备独有的，它详细描述了设备的色彩特性，包括色域大小、白点、对比度等。通过使用ICC配置文件，系统和应用程序可以准确地解释和转换色彩，确保在不同设备上呈现一致的视觉效果。我认为这是正确的导出图片的方法，这样不仅在广色域手机上你的图片可以正常显示，来到了过时的老手机上，你的图片也会被正确地转换回sRGB等等色彩空间。

尾声

我写完了么？感觉也没写完，但我要结束这个文章了。

至少在Observer、gamma、白电平黑电平、手机和Windows的色彩管理逻辑、校色仪原厂校色软件、打印机校色等等地方，由于资料不足和个人水平有限没能继续下去。

谨以此文纪念这个困扰我长达三年的问题，由于网络中没有一个准确的描述，还净是错的，我只能自己摸索。同时也感谢张展同学借我的爱色丽，比红蜘蛛好用多了！！

在文章的结尾，我也想再次提醒入坑色彩管理的各位发烧友：不要迷信参数、不要东施效颦（不理解原理盲目按照他人设置）、不要过于依赖机器，只有自己的眼镜看见的正确才是真的正确，校色后效果不佳大概率是流程中出了问题，不要怀疑自己对色彩的感知，最后，希望每个人能拥有自己理想的显示效果，谢谢您的观看。

拍出来不一样，实际看着一样，这就是Observer参数起的作用。校色仪也是拜耳滤镜，它是摄像头，不是你的眼睛。
